## 🔑 可能被问到的问题

### Q: 为什么选择 Vue 3 而不是 React？
A: Vue 3 的 Composition API 更适合逻辑复用，`<script setup>` 语法简洁，团队对 Vue 更熟悉。

### Q: 路由权限是怎么做的？
A: 在 router/index.js 的 beforeEach 守卫中，根据用户角色（admin/teacher/student）自动重定向。

### Q: API 请求是怎么封装的？
A: 统一在 eduevalApi.js 中封装，支持多 baseURL fallback，统一错误处理。

### Q: 状态管理用了什么？
A: Pinia，比 Vuex 更轻量，TypeScript 支持更好。

### Q: "记住我"功能是怎么实现的？
A:
- 用 localStorage 和 sessionStorage 配合
- 勾选"记住我" → Token 存 localStorage（持久化）
- 不勾选 → Token 存 sessionStorage（会话级）
- 用 edueval_auth_remember 标记用户选择
- 页面加载时自动检查并恢复状态
