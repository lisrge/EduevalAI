# lzc 文件夹博客摘要（1-4）

## 个人工作总结（lzc）

lzc 主要负责后端框架基础设施与数据层能力建设，并在此基础上实现“工作日志/博客”这一独立模块的后端与管理能力，确保后端具备可持续迭代的工程底座。

- 技术理解：用 FastAPI 的路由模块化把接口按领域拆分（认证/用户/申报/导出等），用 Pydantic Settings 统一管理环境配置，避免硬编码；用 SQLAlchemy ORM + Session 把数据库访问收敛为可控的工作单元，保证一次请求内的写入提交与错误回滚一致。
- 已完成进度：完成后端入口与跨域配置；完成数据库连接、建表与核心模型/Schema 约束；完成用户与会话的分离建模；实现迷你博客系统的“发布/存储/渲染”链路，并补齐管理员侧的查看、删除、搜索与权限校验等管理功能。
- 关键实现思想：权限初始化用“首个用户自动成为管理员”降低部署门槛；内容展示采用“Markdown 原文 + HTML 缓存”以减少重复转换开销；管理能力通过“接口语义化（例如 DELETE）+ 权限判定”保证安全边界清晰。

## 博客 1（lzc-1.md）摘要

- 主题：后端服务入口与基础设施搭建，为全系统提供配置管理与数据持久化底座。
- FastAPI 入口：创建应用实例、配置 CORS、注册健康检查/认证/用户/申报/导出等路由模块，形成可维护的模块化 API 结构。
- 数据层支撑：引入 SQLAlchemy 数据库连接与 Session 管理，配合数据库初始化与建表，让业务模块可以稳定读写数据。
- 配置管理：使用 Pydantic Settings 从环境与 .env 加载配置，并支持类型转换与路径拼接，避免硬编码散落各处。
- 数据模型与校验：定义用户/会话/申报/评分结果等模型与 Schema 校验，为后续认证与业务流程提供结构化数据约束。

**专有名词解释（含本项目作用）**
- FastAPI：Python Web 框架；在本项目中承载所有后端 API（认证、申报、导出等），并自动生成接口文档便于联调。
- CORS：跨域资源共享机制；在本项目中解决前后端分离部署时浏览器的跨域限制问题。
- CORSMiddleware：FastAPI 中间件；在本项目中为响应添加跨域相关头部，允许前端从不同域访问后端接口。
- 路由（router）：接口模块的集合；在本项目中按认证/用户/申报/导出拆分，降低耦合并便于维护。
- SQLAlchemy ORM：对象关系映射；在本项目中用 Python 类映射数据库表，减少手写 SQL 并统一数据访问风格。
- Session（数据库会话）：数据库连接上的工作单元；在本项目中用于管理一次请求内的查询/写入与提交回滚。
- Pydantic Settings：配置管理方案；在本项目中统一读取端口、数据库地址、跨域白名单与存储路径等运行参数。
- Schema（Pydantic 数据模型）：请求/响应的数据结构约束；在本项目中用于校验注册登录与申报数据，减少脏数据进入业务层。

## 博客 2（lzc-2.md）摘要

- 主题：核心数据模型拆分与安全设计，重点说明用户信息与会话状态的分离建模。
- 模型拆分：User 存静态信息（学号、密码哈希、签名等）；UserSession 存动态状态（token 哈希、登录时间、过期时间），支持多设备登录与强制下线等扩展。
- 安全考虑：密码与 token 分离；token 仅存哈希；密码采用 salt + hash 的思路存储，降低泄露风险。
- ORM 关系：通过一对多关系把用户与会话关联起来，并配置级联删除，便于用户删除时清理会话记录。
- 辅助细节：使用前向引用解决类型引用问题；用 Optional 表示可空字段；统一时间戳字段用于审计。

**专有名词解释（含本项目作用）**
- User / UserSession：用户表与会话表；在本项目中分别承载“身份资料”与“登录状态”，让认证与权限判断更清晰。
- token_hash：会话 token 的哈希值；在本项目中用于验证请求身份但不保存明文 token，降低数据库泄露风险。
- cascade（级联删除）：关联数据自动清理规则；在本项目中用于删除用户时自动删除其会话等关联记录。
- ForeignKey（外键）：表之间的关联约束；在本项目中用于把会话记录绑定到用户，便于按用户查询与清理。
- Salt + Hash：密码存储方案；在本项目中用于安全存储密码（不存明文），并避免相同密码产生相同哈希。
- 前向引用（future annotations）：延迟解析类型注解；在本项目中用于 ORM 关系类型标注，避免循环引用导致的导入问题。

## 博客 3（lzc-3.md）摘要

- 主题：实现一个独立的“迷你博客/工作日志系统”，支持 Markdown 日志发布与管理员查看管理。
- 技术选型：FastAPI 作为 API 框架；SQLite + SQLAlchemy 作为轻量持久化；Markdown 库负责 MD 转 HTML；认证采用 JWT 思路与哈希加密。
- 数据库设计：用户表以 12 位学号为唯一标识；日志表保存 MD 原文与转换后的 HTML，并用外键关联作者，便于查询与渲染。
- 功能实现：注册/登录、发布日志（Form 表单接收参数、MD 转 HTML 后入库）、修改密码等主流程。
- 性能与体验：保存 HTML 作为渲染缓存，前端展示时可直接渲染而不必每次转换。

**专有名词解释（含本项目作用）**
- SQLite：嵌入式数据库；在本项目中作为轻量存储方案，便于开发与部署（一个文件即可）。
- Jinja2：模板引擎；在本项目的博客系统里可用于生成页面（若采用服务端渲染）或辅助输出 HTML。
- Markdown → HTML：格式转换；在本项目中用于把同学上传的工作日志（MD）转换为可直接展示的 HTML。
- 外键关联：作者与日志的关系；在本项目中便于按用户维度查询日志与统计数量。
- JWT：一种 token 认证方案；在本项目中用于描述“登录后携带 token 访问受限接口”的认证思路。
- OAuth2 表单认证：常见登录表单规范；在本项目中用于实现登录接口的表单提交与 token 返回。

## 博客 4（lzc-4.md）摘要

- 主题：管理员后台能力建设，包括用户管理、日志管理、搜索与退出登录等管理功能。
- 管理员策略：首个注册用户自动成为管理员（is_admin 标识），简化管理员初始化流程。
- 权限校验：管理员可查看所有用户与日志；删除日志时校验“作者本人或管理员”才能删除，普通用户不能删除他人内容。
- 管理功能：提供日志删除 API（DELETE）、用户搜索（按学号/用户名模糊匹配）、退出登录等能力，支持后台管理闭环。
- 账号管理：文档中提出保存明文密码便于管理员查看的方案，并描述注册/修改密码时同步更新（此点在实际项目中应谨慎对待）。

**专有名词解释（含本项目作用）**
- is_admin：管理员标识字段；在本项目中用于区分普通用户与管理员权限，控制后台管理入口与敏感操作。
- DELETE 接口：HTTP 方法之一；在本项目中用于实现日志删除等“破坏性操作”，语义清晰。
- 权限验证：对操作是否允许的判断；在本项目中用于保护删除/查看全量数据等敏感操作。
- 模糊匹配：非完全相等的查询；在本项目中用于后台按学号/用户名搜索用户，提升定位效率。
- 明文密码：未加密的密码文本；在本项目中被用作“管理员查看”的设计讨论点，但从安全角度应尽量避免并改用重置流程替代。
