
# EduevalAI 前端答辩讲解稿

## 开场白（约1分钟）

大家好，我负责的是 EduevalAI 教育评估平台的前端开发工作。

EduevalAI 是一个面向高校课程的智能作业评估系统，主要解决传统作业批改中效率低、工作量统计难等问题。前端作为用户与系统交互的窗口，承担着学生提交作业、教师评分、管理员管理等核心功能。

接下来我从技术选型、核心架构、主要功能、前后端协作这几个方面来介绍。

---

## 一、技术选型（约1分钟）

前端采用的是 **Vue 3 + Pinia + Tailwind CSS** 的技术栈：

1. **Vue 3**：使用 Composition API 和 `<script setup>` 语法糖，代码更简洁、逻辑复用性更强
2. **Pinia**：Vue 3 官方推荐的状态管理库，比 Vuex 更轻量，TypeScript 支持更好
3. **Tailwind CSS**：原子化 CSS 框架，快速构建响应式界面
4. **Vue Router 4**：处理路由和权限守卫

这些技术都是目前 Vue 生态中的主流选择，社区活跃、文档完善，能够快速支撑项目开发。

---

## 二、核心架构设计（约1.5分钟）

前端项目采用了清晰的分层架构：

### 1. 目录结构
```
src/
├── views/       # 页面层 - 对应不同路由的页面组件
├── components/  # 组件层 - 可复用的UI组件
├── services/    # 服务层 - 封装所有后端API调用
├── stores/      # 状态层 - Pinia 状态管理
├── router/      # 路由层 - 路由配置和权限守卫
└── utils/       # 工具层 - 通用工具函数
```

### 2. 关键设计

**API 服务层** (`eduevalApi.js`)：
- 统一封装所有后端接口
- 支持多 baseURL 自动 fallback（当前端部署在不同环境时自动适配）
- 统一的错误处理和响应解析

**路由权限守卫** (`router/index.js`)：
- 基于用户角色自动重定向
  - 学生 → 作业提交页
  - 教师 → 评分端
  - 管理员 → 用户管理
- 未登录用户拦截到登录页

**状态管理** (`stores/authStore.js`)：
- 管理用户认证状态
- **重点："记住我"功能的实现**

  这个功能是通过 `localStorage` 和 `sessionStorage` 配合实现的：

  1. **存储策略**：
     - 勾选"记住我" → Token 存入 `localStorage`（持久化，关闭浏览器不丢失）
     - 不勾选 → Token 存入 `sessionStorage`（会话级，关闭浏览器后清除）
     - 同时用一个标记 `edueval_auth_remember` 记录用户选择

  2. **初始化流程**：
     - 页面加载时调用 `ensureInitialized()`
     - 先检查存储中是否有 Token
     - 如果有，自动调用后端 `/auth/me` 接口验证并恢复用户状态
     - 如果 Token 失效，自动清除

  3. **安全考虑**：
     - 每次存储前先清空两个 Storage，防止冲突
     - 登出时同时清除两个 Storage 中的 Token 和标记

## 三、核心功能实现（约1.5分钟）

我重点实现了两个核心业务页面：

### 1. 学生作业提交页 (`HomeworkCollectionView.vue`)

**功能特点**：
- 左侧：作业列表，显示课程、周次、提交状态
- 右侧：分模块填写提交内容
  - 基本信息：姓名、小组、项目名
  - 成员管理：支持添加/删除成员，每人填写学号、角色、个人陈述
  - 材料上传：支持报告、代码包、PPT、视频等多种类型

**交互流程**：
可以先保存草稿，随时回来修改，确认无误后再"最终提交"

### 2. 教师移动评分端 (`TeacherScoringMobileView.vue`)

**设计亮点**：
- 响应式布局，支持手机端评分
- 左侧队列：可按作业筛选、待评/已评筛选
- 右侧评分详情：
  - 查看提交内容、成员工作量分析
  - 5个评分维度：创新性、完整性、代码质量、演示效果、个人贡献
  - 填写评语后一键提交，自动进入下一份

## 四、前后端协作（约0.5分钟）

后端 API 运行在 `http://localhost:8001/api`，前端通过 `fetch` 调用：

- 所有请求都在 `eduevalApi.js` 中统一封装
- 认证通过 Bearer Token 实现
- 主要接口：
  - `/auth/login` - 登录
  - `/assignments` - 获取作业列表
  - `/assignments/{id}/submissions` - 提交作业
  - `/teacher/submissions` - 教师评分队列
